## Question
What is the relationship between a client program and a server program?

## Answer
Based on Beej's Guide to Network Programming, the relationship between a client and server is defined by **asymmetry** and **initiation**.

Here are the key aspects of that relationship:

### 1. The Initiator vs. The Listener
*   **The Server** is passive. It starts first, binds to a specific well-known port, and then enters an infinite loop waiting (listening) for incoming connection requests. It does not initiate contact.
*   **The Client** is active. It starts after the server is running, knows the server’s IP address and port, and initiates the connection request.

### 2. Service Provider vs. Service Consumer
*   The **Server** provides a specific service (e.g., serving web pages, handling email, storing files). It exposes resources or functionality.
*   The **Client** consumes that service. It sends requests and receives responses. A single server typically serves many clients simultaneously.

### 3. Addressing Knowledge
*   The **Client must know the server’s address** (IP + port) beforehand. This is often hardcoded, configured, or resolved via DNS.
*   The **Server does not need to know the client’s address** in advance. It learns the client’s address only when the connection is established, and it can serve any client that reaches it.

### 4. Lifecycle Independence
*   Servers are generally **long-running** processes designed to stay up indefinitely.
*   Clients are typically **short-lived** — they connect, complete a task, and disconnect. The server continues running regardless of whether any particular client is connected.

### Beej’s Key Insight
Beej emphasizes that **"client" and "server" are roles, not inherent properties of machines**. The same physical computer can run both client and server programs simultaneously. What matters is *which program initiated the connection*. As Beej puts it, the server is the one that says "I'm here, talk to me," and the client is the one that says "OK, I'd like to talk."

> ⚠️ **Important:** This client-server relationship exists at the **application layer**. At lower layers (IP, Ethernet), communication is peer-to-peer — packets don't know or care which end is "client" or "server." The asymmetry is purely a software/application-level convention.
